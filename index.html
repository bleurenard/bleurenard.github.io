发表于 2019-08-07

历经五个小时的奋斗，终于把个人博客搭建好了，期间采坑无数，谨以此篇记录一路辛酸泪。

first

找教程！！！

我现在发现找篇靠谱的教程是在是太重要了，网上大多数教程都是相似的这就存在了一个问题,一旦其中出现了问题，那么大片教程就都废了，我在配置环境时就遇到了一个问题，大多数教程在安装hexo时都建议

先将npm的源设置成淘宝镜像站

npm config set registry "https://registry.npm.taobao.org"

再在git bash中执行

npm install hexo-cli g

但尝试了很多次都没能成功，所以果断换了一篇博客

http://visugar.com/2017/05/04/20170504SetUpHexoBlog/

该博客安装hexo时采用

npm i -g hexo

经博主验证可行。

其次该篇博客内容也较为充实，不仅仅记录了如何搭建环境，而且包括了一些辅助功能的设置，极为良心。

second

由于上一部分的链接中有较为详细的安装过程，所以在这一部分就只是简单介绍一下搭建过程，和期间一些知识盲区。

安装步骤：

  1. ​ 安装Git Bash（官网下载，免费无害）

  2. 安装NodeJs（同样官网下载，注：LTS为长期支持版，Current为最新版）

  3. 安装hexo（Git Bash里敲语句）

  4. 生成SSH并添加到GitHub（做这一步主要因为git使用https协议，每次pull、push都要输入密码，相当烦。使用SSH密钥可以省去每次都输密码）

  5. 部署项目（直接修改blog文件下的_config.yml文件，就是这么简单粗暴）

  6. 上传GitHub（hexo三板斧hexo clean hexo generate hexo deploy)

    hexo generate 生成静态文件
